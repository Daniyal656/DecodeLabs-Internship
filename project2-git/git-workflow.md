# Git Workflow

## The Three Trees 
- Working Tree: Where you write code (sandbox)
-  Staging Area: Where you prepare commits (loading dock)
- Repository: Permanent committed history (ship's log)

## Basic Workflow
Working Tree → git add → Staging Area → git commit → Repository → git push → GitHub

## Branching Strategy
- main: production ready code only
- feature branches: experimental work
- Always merge via pull request
- Never break production
